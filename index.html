What it is · 개요

Three ideas hold the whole system up

이 시스템을 떠받치는 세 가지 핵심 아이디어

Every source — a YouTube clip, a world-model rollout, a sim trace, a teleop demo — emits the same canonical record. Differences in trustworthiness travel as metadata, not as separate code paths.

어떤 출처든 — 유튜브 클립, 월드모델 롤아웃, 시뮬레이션 트레이스, 텔레오프 시연 — 똑같은 표준 레코드를 내보냅니다. 신뢰도 차이는 별도의 코드 경로가 아니라 메타데이터로 따라붙어요.

①

Manifest-first

Layers exchange small, typed manifests plus immutable artifact URIs — never raw frames or tensors. A control plane drives execution; a data plane holds the bytes. Each layer is independently testable and portable from laptop to cloud.

매니페스트 우선. 각 단계는 작은 타입드 매니페스트와 불변 아티팩트 URI만 주고받습니다 — 원본 프레임·텐서를 직접 넘기지 않아요. 실행을 지휘하는 컨트롤 플레인과 바이트를 담는 데이터 플레인이 분리돼, 각 단계를 따로 테스트하고 노트북에서 클라우드까지 그대로 옮길 수 있습니다.

②

Trainability is computed

Each episode carries a capability vector (has_video, has_state, has_action, license, consent…). Its trainability level is derived from that vector + provenance + validation — never inferred from the source type alone.

학습 등급은 계산된다. 각 에피소드는 능력 벡터(영상·상태·액션 유무, 라이선스·동의 등)를 갖고, 학습 등급은 그 벡터 + 출처 + 검증 결과로 계산됩니다 — "어디서 왔는지(출처 종류)"만 보고 정하지 않아요. 멋져 보이는 출처라도 데이터가 받쳐주지 않으면 등급이 올라가지 않습니다.

③

The writer is a materializer

The internal canonical form stays more expressive than any LeRobot version. The writer materializes it into a v3 chunked dataset. Action namespaces (robot, sim, pseudo_eef, latent) stay distinct — never collapsed or upcast.

라이터는 구체화기다. 내부 표준 형식은 어떤 LeRobot 버전보다 표현력이 큽니다. 라이터는 그걸 v3 데이터셋으로 구체화(materialize)할 뿐이에요. 액션 네임스페이스(robot·sim·pseudo_eef·latent)는 출처별로 분리된 채 유지되고, 절대 하나로 합치거나 등급을 올리지 않습니다.

The trainability ladder · 학습 등급 사다리

The central invariant: never silently upcast

핵심 불변 규칙: 절대 몰래 등급을 올리지 않는다

Every episode resolves to a level 0–4, computed from its capability vector. Video without trustworthy robot state/action is not policy-training data — regardless of how impressive the source looks. Our demo clip is video-only, so it lands on rung 1 and validation blocks the policy route.

모든 에피소드는 능력 벡터로 0~4 등급 중 하나로 계산됩니다. 핵심 질문은 단 하나 — "믿을 수 있는 로봇 상태/액션이 있는가?"예요. 신뢰할 수 있는 상태·액션이 없는 영상은, 출처가 아무리 그럴듯해도 정책(policy) 학습 데이터가 아닙니다. 이 데모 클립은 영상만 있어서 1등급에 놓이고, 검증이 정책 학습 경로를 막아요.

등급은 어떻게 구분되나 · How levels are decided

등급은 선언하는 게 아니라 능력 벡터로 계산됩니다. 기준은 "어떤 액션 신호가, 얼마나 믿을 수 있게 있는가":

  • L0 archive_only — 쓸 수 있는 게 없거나 법적/동의 차단 → 보관만
  • L1 video_only — 영상(픽셀)만 → 표현·사전학습
  • L2 pseudo_action — 상태(state)나 영상 추정(pseudo) 액션 → 의사-액션 학습
  • L3 sim_action_policy — 실제 액션+상태(sim 또는 미보정 real) → 정책 사전학습
  • L4 real_robot — 보정된 real(임베디먼트+카메라 캘리브레이션) → 실로봇 지도학습

"몰래 올리지 않는다"는 것 · Never silently upcast

믿을 수 없는 데이터를 더 높은 등급인 척 쓰면 학습이 통째로 오염됩니다. 그래서 세 가지가 강제돼요:

  • 검증(validation)은 등급을 내릴 수만 있고, 절대 올리지 못함
  • 높은 등급을 주장해도 실제 action·state 파일이 디스크에 없으면 차단 — 위조된 매니페스트는 통과 못 함
  • 액션 네임스페이스(robot·sim·pseudo_eef·latent)를 절대 합치지 않음 — pseudo/latent를 real 채널에 섞는 건 영상-only를 정책 데이터로 올리는 것과 똑같은 위반

Where it's going · 앞으로의 방향

Same contracts, three phases

동일한 계약(인터페이스), 세 단계의 로드맵

The layer boundaries already speak manifests + artifact URIs. Each phase is a mechanical step up from the last — swap the filesystem for S3, the loop for a workflow engine, the SQLite ledger for Athena. The contracts don't change.
